# スキル自動生成ツール

name: skill-creator
description: 汎用的な作業パターンを発見した際に、再利用可能なClaude Codeスキルを自動生成する。繰り返し使えるワークフロー、ベストプラクティス、ドメイン知識をスキル化。

## Overview

メンバーや部長が繰り返し行う作業パターンを発見した際に、それを再利用可能なスキルとして保存するためのツール。

## スキル化フロー

```
メンバーがスキル化候補を発見
    ↓ 報告
部長に reports で報告（skill_candidate フィールド）
    ↓
部長会議で共有（agenda に記載）
    ↓
議長が設計書を作成
    ↓
dashboard.md の「スキル化候補」に記載
    ↓
社長が承認
    ↓
議長が skill-creator を使用してスキル作成
    ↓
skills/ ディレクトリに保存
```

## スキル化候補の判断基準

以下のいずれかに該当する場合、スキル化を検討:

1. 他のプロジェクトでも使えそうな汎用的なパターン
2. 同じパターンを2回以上実行した
3. 他のメンバーにも有用そう
4. 特定の手順や知識が必要な作業
5. エラーが起きやすく、手順の標準化が有効

## スキル構造

```
skill-name/
├── SKILL.md          # 必須: スキル定義
├── scripts/          # オプション: 実行スクリプト
└── resources/        # オプション: 参照ファイル
```

## SKILL.md テンプレート

```markdown
# スキル名

name: kebab-case-name
description: いつ使うか、具体的なユースケースを記述。Claudeがこの説明を見てスキルを使うか判断する。

## Overview

スキルの概要説明。

## When to Use

このスキルを使うべき状況:
- トリガーとなるキーワード
- 具体的なシナリオ

## Instructions

### Step 1: 〇〇
具体的な手順

### Step 2: 〇〇
具体的な手順

## Examples

### 入力例
```
ユーザーの入力例
```

### 出力例
```
期待される出力
```

## Guidelines

- 守るべきルール
- 注意点
- 禁止事項
```

## スキル名の命名規則

- kebab-case を使用（例: api-error-handler）
- 動作を表す動詞 + 対象を表す名詞（例: format-meeting-notes）
- 短く分かりやすい名前

## 良いdescriptionの例

```
name: meeting-notes-formatter
description: 議事録を標準フォーマットに変換する。参加者、決定事項、アクションアイテムを抽出・整理。会議後のドキュメント作成時に使用。
```

```
name: api-error-handler
description: APIエラーレスポンスを標準フォーマットで生成する。エラーコード、メッセージ、詳細情報を含む。REST API開発時のエラーハンドリングに使用。
```

## 保存先

作成したスキルは以下に保存:
```
~/.claude/skills/company-{skill-name}/SKILL.md
```

または、プロジェクト内に保存:
```
skills/{skill-name}/SKILL.md
```

## 使用方法（議長向け）

1. 社長の承認を得る
2. スキル設計書を確認
3. 上記テンプレートに従ってSKILL.mdを作成
4. 必要に応じてscripts/やresources/を追加
5. 既存スキルとの重複がないか確認
6. 保存先に配置
7. dashboard.md を更新（スキル化完了を記載）

## 注意事項

- description の記述が最も重要（Claudeがスキルを使うか判断する材料）
- 既存スキルと名前の重複を必ず確認
- 汎用性を意識しつつ、具体的な手順を記載
- 最新の仕様やベストプラクティスをリサーチしてから作成
